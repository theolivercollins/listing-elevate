**Spec:** `docs/specs/2026-05-23-v1.1-seedance-pushin-design.md`

## Parallel partitioning

Each lane owns its files. No file is shared across lanes, so the four worker subagents can run concurrently without merge conflict. The coordinator runs the wire-up + smoke after all four return.

**Deliverables:**

Pin the Seedance model version SHA — fetch the latest from `https://api.replicate.com/v1/models/bytedance/seedance-1-pro` and bake the version id as a constant (`SEEDANCE_VERSION_SHA`).
- Cost: `SEEDANCE_CENTS_PER_SECOND = 12` constant in the file. `checkStatus` returns `costCents = durationSeconds * 12` on success (Replicate doesn't echo prediction cost in the response body).
- Tests: mock `fetch`, cover happy-path submit, status mapping (`starting`/`processing` → `processing`, `succeeded` → `complete` with `output[0]`, `failed`/`canceled` → `failed`), cost math, REPLICATE_API_TOKEN missing → constructor throws.

**Acceptance:** `pnpm test lib/providers/__tests__/seedance.test.ts` green.

### Lane B — Speed-ramp FFmpeg utility

**Files:**
- EDIT `lib/utils/ffmpeg.ts` — add `applySpeedRamp` + `applySpeedRampToBuffer`.
- NEW `lib/utils/__tests__/ffmpeg.speed-ramp.test.ts`.

**Deliverables:**
- `applySpeedRamp(inputPath, outputPath, opts?)` runs `ffprobe` to read duration, builds the 3-segment `trim`/`setpts`/`concat` filter, execs ffmpeg, throws on non-zero exit.
- `applySpeedRampToBuffer(buf, opts?)` writes buffer to `os.tmpdir()/seedance-ramp-${uuid}.mp4`, calls `applySpeedRamp`, reads result, cleans both files via `try/finally`.
- Defaults: `rampSeconds = 0.5`, `rampFactor = 0.8`.
- Guard: if `inputDuration < 2 * rampSeconds + 0.1`, throw `Error('clip too short for speed ramp')` (caller in poll-scenes will catch and log + fall back to original buffer).
- Test: generate a 5-second testsrc clip with `ffmpeg -f lavfi -i testsrc=duration=5:size=320x240:rate=30`, run `applySpeedRamp`, ffprobe the output, assert duration is within 50ms of `5 + 2 * 0.5 * (1/0.8 - 1) = 5.25s`.

**Acceptance:** `pnpm test lib/utils/__tests__/ffmpeg.speed-ramp.test.ts` green. Test skips on CI with a clear message if `ffmpeg` is not on PATH (the existing tests do this — copy the pattern).
