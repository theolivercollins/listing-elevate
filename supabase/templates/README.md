# Email templates

`magic-link-otp.html` is the Supabase Auth email template for the **"Magic link or OTP"** slot — paste its full contents into Supabase Dashboard → Authentication → Emails → Magic Link (Source view), replacing whatever is there now; it renders both the `{{ .ConfirmationURL }}` sign-in button and the `{{ .Token }}` 6-digit code needed by the admin email-code step-up (`AdminEmailVerifyWall`) — leave those Go-template variables exactly as written, Supabase substitutes them server-side. `welcome.html` is the transactional "Welcome to Listing Elevate" email for new signups. As of 2026-07-01 it **is wired to a sender**: `lib/email/welcome-template.ts` inlines a byte-for-byte copy of this file (see that file's header comment for why it isn't read off disk at runtime), and `api/hooks/welcome-email.ts` sends it via Resend when a Supabase Database Webhook fires on `auth.users` INSERT. If you edit the design here, copy the new markup into `lib/email/welcome-template.ts` in the same commit — there is no automated check keeping them in sync. Off by default (`WELCOME_EMAIL_ENABLED` unset); see that file's header for the env vars + webhook config needed to turn it on. Both files are self-contained, inline-styled, table-based HTML (max-width 600px) built to survive Gmail/Outlook/Apple Mail clipping and stripped `<style>` blocks; do not add external CSS/JS or webfont `<link>` tags — the Inter/system-font stack is intentional (no monospace, per project rule).
